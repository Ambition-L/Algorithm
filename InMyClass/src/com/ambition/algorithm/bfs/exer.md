## 艰难旅行思路

### 规则
```text
给一组由二维数组组成的0 1矩阵 和原点坐标 输出最大行走格子数
（限定 不同坐标值不同才能移动 只能上下左右移动且不能越界）
```

### bsf思想
```text
中心扩散思想
    1、由原点向四周扩散 利用队列的思想 将出队的坐标进行扩散 且记录坐标状态
    2、故第一步先创建队列
    3、将原点坐标放入队列中
        3.1 由于坐标包含 x y 和 坐标值 故使用引用数据类型来保存坐标信息 （类类型）
        3.2 队列类型为类类型（Grid） 由于二维数组的索引从0，0开始 故坐标值为 x-1，y-1
    4、创建hashmap数据结构保存坐标状态 
        4.1 key为 （x + "." + y）确保坐标唯一性 故最后map保存了所有访问到的格子坐标 那么map的长度就是格子的个数
        4.2 value 为状态 1 和 2 ； 1代表入队 2代表出队 故原点初始值为1
    5、准备阶段完毕之后开始由原点扩散
        5.1 出队操作：出队的坐标就是要扩散的坐标 只要队列不为空就说明还有格子未被扩散
        5.2 坐标移动的条件 
            01、行列移动不越界
            02、移动后的坐标值不等于移动前坐标值 （0-1）（1-0）   
            03、未被访问过 （hash表中不存在该坐标）
        5.3 满足以上条件才有扩散的必要
            01 可以扩散但是要判断是否已经扩散 （判断是否在队列中--遍历链表 比较x，y坐标）
                如果在则跳出循环
            02 如果不在则加入队列 并且添加入状态记录表中 状态设置为1
        5.4 再一次向左右下移动 （一次进行5.2 -> 5.3 -> 5.1）  
    6、直到队列中没有数据 （说明已经没有满足扩散条件的坐标）
        输出map长度就是所有可访达的格子数量          
```
